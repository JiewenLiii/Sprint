# AI 辅助分析备忘录

**项目名称**：C++ 控制台文字地牢探险游戏  
**Sprint 周期**：2 周 (MVP 版本)  
**团队角色**：产品 + 架构顾问（AI 辅助）

## 1. 项目可行性结论

### ✅ 可行性依据

| 维度 | 分析 |
|------|------|
| **技术栈** | 仅使用 C++ 标准库（类、字符串、控制台 I/O、随机数），无第三方依赖 |
| **开发难度** | 2 周可完成核心闭环：进入游戏 → 探索地牢 → 遭遇敌人 → 回合战斗 → 胜利/失败 |
| **团队协作** | 功能可清晰拆分为模块（玩家、地图、战斗、输入、UI），适合多人分工 |
| **部署成本** | Windows/Linux/macOS 均可直接编译运行，零部署成本 |

### 📌 结论
**项目可行**，建议按 MVP 范围严格执行，避免范围蔓延。

## 3. 核心玩法逻辑

### 状态流转图
```
开始 → 生成地图 → 放置玩家 → 放置敌人 → [循环开始]
                                        ↓
                    ┌───────────────────┘
                    ↓
                等待输入 → 处理输入 → 更新状态 → 渲染
                    ↑                          ↓
                    └────── 判断结束条件 ──────┘
                               ↓
                            是 → 结束
```

### 核心循环
1. **渲染**：绘制地图、玩家 (@)、敌人 (E)
2. **输入**：WASD 移动，Q 退出
3. **更新**：玩家移动、敌人追击
4. **战斗**：接触触发回合制战斗
5. **判定**：HP=0 失败，全灭敌人胜利

## 4. 可能遇到的卡点与应对方案

### (1) 产品卡点

| 卡点 | 表现 | 应对方案 |
|------|------|----------|
| 范围蔓延 | "再加个装备系统吧" | 严格对照 MVP 清单，新增需 PO 批准 |
| 难度平衡 | 太难/太简单 | 硬编码可调参数（攻击/血量），便于调整 |
| 体验反馈 | 不知道发生了什么 | 每回合显示战斗日志（"你攻击了哥布林 -3HP"） |

### (2) 技术卡点

| 卡点 | 表现 | 应对方案 |
|------|------|----------|
| 输入缓冲 | `cin>>` 后 `getline` 失效 | 统一用 `cin>>`，或 `cin.ignore()` 清理缓冲 |
| 控制台闪烁 | `system("cls")` 闪屏 | 用 `SetConsoleCursorPosition` 替代（可选优化） |
| 碰撞检测 bug | 穿墙/重叠 | 封装统一 `canMove(x,y)` 检查函数 |
| Windows 编码 | 中文乱码 | 文件保存为 UTF-8，或用 `SetConsoleOutputCP(65001)` |

## 6. 技术选型建议

### 编译环境
- **标准**：C++11 或更高
- **Windows**：MinGW g++ 或 MSVC
- **Linux/macOS**：g++ / clang++

### 代码规范
- 模块化设计：`.h` + `.cpp` 分离
- 命名规范：类名大驼峰，函数名小驼峰
- 注释：关键函数添加 Doxygen 风格注释

### 版本控制
- 主干保护：禁止直接 Push 到 main/master
- PR 审查：至少 1 人 Review + Approve
- 分支策略：`feature/xxx` 分支开发
